# 階段 0：資料集整理任務

骨架已建好，接下來由你擴充。**這是練習的一部分**——整理資料時留意：
哪些欄位是給工具做結構化查詢用的（hours、lat/lng、price_range），
哪些是給 RAG 語意檢索用的（docs/ 裡的介紹文）。

## 你的任務清單

- [ ] 把 `shops.json` 擴充到 **20 家以上**（目前 10 家）
  - 台南、台中各保持至少 5 家（階段 1 用台南、階段 2/4 用台中）
  - 建議加第三個城市（例如高雄或嘉義），階段 4 排兩天一夜時更有變化
- [ ] 把 5 筆 `verified: false` 的條目查證：找到食尚玩家的報導連結填入 `source`
  - 查證入口：https://supertaste.tvbs.com.tw/search/ （搜店名）
- [ ] 每家店寫一篇 `docs/<店名>.md`，100–300 字（目前 3 篇，格式照現有範例）
  - 內容盡量寫「只有看過介紹才知道」的細節（誰推薦、招牌吃法、限量資訊）——
    這些細節之後是驗證 RAG 有沒有真的檢索到文件的關鍵
- [ ] 跑驗證：`python3 scripts/validate_data.py`，錯誤清零、提醒盡量清掉

## 欄位說明

| 欄位 | 用途 | 格式 |
|---|---|---|
| hours | 階段 2 `is_open` 工具 | `HH:MM-HH:MM`，跨夜寫 `00:00-07:00` 或 `18:00-00:30` |
| lat/lng | 階段 2 `distance` 工具 | Google Maps 上點右鍵可複製座標 |
| price_range | 階段 4 預算控管 agent | `低-高`（新台幣） |
| note / docs/*.md | 階段 3 RAG 檢索 | 自由文字 |
| verified/source | 資料出處紀錄 | 查證後改 true 並附連結 |
